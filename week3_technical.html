Ruby Basics: Arrays v. Hashes

This week for the technical blog post I am going to be focusing on Ruby. Not only that, but I am going to breakdown for you the dynamics of arrays and hashes. I will go over what they are, what they are good for and how to determine which one to use. Also, I will explain a strategy to help you make the decision of which to implement into your code.

Arrays

In a nutshell, an array is a collection of objects. These objects can be strings, integers, etc. The thing that makes an array useful is that it acts as a storage system, that you can call the data contained within based on its numerical position in the array. For example:

For example, by calling array[0]; you will be able to access the first value in the array, in this case "Joe". It is important to remember, that the position values start counting from 0 and not 1. Arrays are ideal for storing unordered, single pieces of information.

Hashes

Hashes, although similar to arrays, are different in a few respects. First, let’s talk about their similarities. Hashes, like arrays, are used to store data, which you can access from outside the hash. However, unlike arrays, hashes use an unordered system of key-value pairings. Then, in order to access the values, you call the information based on the key rather than a numerical place.

All This is Great, But Which Do I Use?

First, you should look at what information you are trying to store.

  1. Does my data need order?
  2. Does it have one or two levels of information?
  3. Do I have a lot of information I need to access easily?

Then from this, based on the descriptions above, I bet you can determine if you need a hash or an array. If your data needs order, you need an array. If you have a lot of data, coming up with an ordering system will make it easier to access later, so you would also need an array. If you only have one level of information, array. If you have two levels, you should use a hash, unless you need the data to have a specific order, then you could create an array. Basically, arrays can be much more useful for larger pieces of data, but if you only need a small two-leveled piece of information, then a hash is more than sufficient.

Why is JavaScript so popular in web development?

JavaScript is so popular in web development for many reasons. The JavaScript language is intended to be used to allow interaction between the user and the website. It uses various processes to make the user experience simpler and therefore more enjoyable. JavaScript has a myriad of applications within modern websites, from autocomplete of searches to telling you your password is incorrect without having to reload the page.

JavaScript can also solve problems that arising when different computers try to access the same webpage. Some computers might load the page and have it look different because of the internet browser they are using. JavaScript can solve this problem by analyzing the browser settings and projecting an image that is clear and organized on the user’s computer. The list of uses could go on and on, but the reason that JavaScript is the most popular web languages for this type of development is that it is one of the easiest to learn in comparison to the others. This creates widespread usage, ultimately adding to its popularity.
